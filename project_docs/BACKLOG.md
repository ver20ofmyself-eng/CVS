### 4. Автоматический анализ из очереди

> **Цель:** При накоплении ≥10 кандидатов (или ≥X) — автоматически анализировать массив всех кандидатов по вакансии, сохранять сводку как дополнение к системному промпту. Оценивать новых кандидатов с учётом текущих.
> **Важно:** Очереди для каждого `user` разные, обрабатываются параллельно.

### 5. Проверить качество анализа при разных объёмах

> Проверить качество анализа при candidates < 10 и > 10. Проработать возможные оптимизации.
